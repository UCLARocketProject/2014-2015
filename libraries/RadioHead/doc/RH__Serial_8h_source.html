39 /// \class RH_Serial RH_Serial.h <RH_Serial.h>
40 /// \brief Driver to send and receive unaddressed, unreliable datagrams via a serial connection
41 ///
42 /// This class sends and received packetized messages over a serial connection.
43 /// It can be used for point-to-point or multidrop, RS232, RS488 or other serial connections as
44 /// supported by your controller hardware.
45 /// It can also be used to communicate via radios with serial interfaces such as:
46 /// - APC220 Radio Data Module http://www.dfrobot.com/image/data/TEL0005/APC220_Datasheet.pdf
47 /// http://www.dfrobot.com/image/data/TEL0005/APC220_Datasheet.pdf
48 /// - 3DR Telemetry Radio https://store.3drobotics.com/products/3dr-radio
49 /// - HopeRF HM-TR module http://www.hoperf.com/upload/rf_app/HM-TRS.pdf
50 /// - Others
51 ///
52 /// The packetised messages include message encapsulation, headers, a message payload and a checksum.
53 ///
54 /// \par Packet Format
55 ///
56 /// All messages sent and received by this RH_Serial Driver conform to this packet format:
57 /// \code
58 /// DLE
59 /// STX
60 /// TO Header (1 octet)
61 /// FROM Header (1 octet)
62 /// ID Header (1 octet)
63 /// FLAGS Header (1 octet)
64 /// Message payload (0 to 60 octets)
65 /// DLE
66 /// ETX
67 /// Frame Check Sequence FCS CCITT CRC-16 (2 octets)
68 /// \endcode
69 ///
70 /// If any of octets from TO header through to the end of the payload are a DLE,
71 /// then they are preceded by a DLE (ie DLE stuffing).
72 /// The FCS covers everything from the TO header to the ETX inclusive, but not any stuffed DLEs
73 ///
74 /// \par Physical connection
75 ///
76 /// The physical connection to your serial port will depend on the type of platform you are on.
77 ///
78 /// For example, many arduinos only support a single Serial port on pins 0 and 1,
79 /// which is shared with the USB host connections. On such Arduinos, it is not possible to use both
80 /// RH_Serial on the Serial port as well as using the Serial port for debugand other printing or communications.
81 ///
82 /// On Arduino Mega and Due, there are 4 serial ports:
83 /// - Serial: this is the serial port connected to the USB interface and the programming host.
84 /// - Serial1: on pins 18 (Tx) and 19 (Rx)
85 /// - Serial2: on pins 16 (Tx) and 17 (Rx)
86 /// - Serial3: on pins 14 (Tx) and 15 (Rx)
87 ///
88 /// On Uno32, there are 2 serial ports:
89 /// - SerialUSB: this is the port for the USB host connection.
90 /// - Serial1: on pins 39 (Rx) and 40 (Tx)
91 ///
92 /// On Maple and Flymaple, there are 4 serial ports:
93 /// - SerialUSB: this is the port for the USB host connection.
94 /// - Serial1: on pins 7 (Tx) and 8 (Rx)
95 /// - Serial2: on pins 0 (Rx) and 1 (Tx)
96 /// - Serial3: on pins 29 (Tx) and 30 (Rx)
97 ///
98 /// Note that it is necessary for you to select which Serial port your RF_Serial will use and pass it to the
99 /// contructor.
