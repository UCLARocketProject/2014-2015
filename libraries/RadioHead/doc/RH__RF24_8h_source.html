488 /// \class RH_RF24 RH_RF24.h <RH_RF24.h>
489 /// \brief Driver to send and receive unaddressed, unreliable datagrams via an RF24 and compatible radio transceiver.
490 ///
491 /// Works with
492 /// - Silicon Labs Si4460/1/2/3/4 transceiver chips
493 /// - The equivalent HopeRF RF24/25/26/27 transceiver chips
494 /// - HopeRF Complete modules: RFM24W/26W/27W
495 ///
496 /// \par Overview
497 ///
498 /// This class provides basic functions for sending and receiving unaddressed,
499 /// unreliable datagrams of arbitrary length to 250 octets per packet.
500 ///
501 /// Manager classes may use this class to implement reliable, addressed datagrams and streams,
502 /// mesh routers, repeaters, translators etc.
503 ///
504 /// Naturally, for any 2 radios to communicate that must be configured to use the same frequency and
505 /// modulation scheme.
506 ///
507 /// This Driver provides an object-oriented interface for sending and receiving data messages with Hope-RF
508 /// RF24 and compatible radio modules, such as the RFM24W module.
509 ///
510 /// The Hope-RF (http://www.hoperf.com) RF24 family is a low-cost ISM transceiver
511 /// chip. It supports FSK, GFSK, OOK over a wide range of frequencies and
512 /// programmable data rates. HopeRF also sell these chips on modules which includes
513 /// a crystal and antenna coupling circuits: RFM24W, RFM26W and RFM27W
514 ///
515 /// This Driver provides functions for sending and receiving messages of up
516 /// to 250 octets on any frequency supported by the RF24, in a range of
517 /// predefined data rates and frequency deviations. Frequency can be set
518 /// to any frequency from 142.0MHz to 1050.0MHz. Caution: most modules only support a more limited
519 /// range of frequencies due to antenna tuning.
520 ///
521 /// Up to 2 RFM24 modules can be connected to an Arduino (3 on a Mega),
522 /// permitting the construction of translators and frequency changers, etc.
523 ///
524 /// The following modulation types are suppported with a range of modem configurations for
525 /// common data rates and frequency deviations:
526 /// - OOK On-Off Keying
527 /// - GFSK Gaussian Frequency Shift Keying
528 /// - FSK Frequency Shift Keying
529 ///
530 /// Support for other RF24 features such as on-chip temperature measurement,
531 /// transmitter power control etc is also provided.
532 ///
533 /// RH_RF24 uses interrupts to detect and handle events in the radio chip. The RF24 family has
534 /// TX and RX FIFOs of 64 bytes, but through the use of interrupt, the RH_RF24 driver can send longer
535 /// messages by filling or emptying the FIFOs on-the-fly.
536 ///
537 /// Tested on Anarduino Mini http://www.anarduino.com/mini/ with arduino-1.0.5
538 /// on OpenSuSE 13.1
539 ///
540 /// \par Packet Format
541 ///
542 /// All messages sent and received by this RH_RF24 Driver conform to this packet format:
543 ///
544 /// - 4 octets PREAMBLE (configurable)
545 /// - 2 octets SYNC 0x2d, 0xd4 (configurable, so you can use this as a network filter)
546 /// - Field containing 1 octet of message length and 2 octet CRC protecting this field
547 /// - Field 2 containing at least 4 octets, and 2 octet CRC protecting this field:
548 /// + 4 octets HEADER: (TO, FROM, ID, FLAGS)
549 /// + 0 to 250 octets DATA
550 /// + 2 octets CRC, computed on HEADER and DATA
551 ///
552 /// \par Connecting RFM-24 to Arduino
553 ///
554 /// For RFM24/RFM26 and Teensy 3.1 or Anarduino Mini
555 /// \code
556 /// Teensy RFM-24/RFM26
557 /// GND----------GND (ground in)
558 /// 3V3----------VCC (3.3V in)
559 /// interrupt 2 pin D2-----------NIRQ (interrupt request out)
560 /// SS pin D10----------NSEL (chip select in)
561 /// SCK pin D13----------SCK (SPI clock in)
562 /// MOSI pin D11----------SDI (SPI Data in)
563 /// MISO pin D12----------SDO (SPI data out)
564 /// D9-----------SDN (shutdown in)
565 /// /--GPIO0 (GPIO0 out to control transmitter antenna TX_ANT)
566 /// \--TX_ANT (TX antenna control in) RFM22B only
567 /// /--GPIO1 (GPIO1 out to control receiver antenna RX_ANT)
568 /// \--RX_ANT (RX antenna control in) RFM22B only
569 /// \endcode
570 /// Caution: tying the radio SDN pin to ground (though it might appear from the data sheets to make sense)
571 /// does not always produce a reliable radio startup. So this driver controls the SDN pin directly.
572 /// Note: the GPIO0-TX_ANT and GPIO1-RX_ANT connections are not required for the 11dBm RFM24W,
573 /// which has no antenna switch.
574 ///
575 /// \par Customising
576 ///
577 /// The library will work out of the box with the provided examples, over the full frequency range and with
578 /// a wide range of predefined modem configurations schemes and speeds. However, you may want to
579 /// change the default behaviour of this library. There are several ways you can do this:
580 ///
581 /// - Use the RH_RF24 API based on this documentation
582 /// - Create your own ModemConfig and pass it to setModemreeegisters()
583 /// - Generate a new radio_config_Si4460.h using the Silicon Labs WDS software package
584 /// - Write directly to the radio registers and properties using command() and set_properties()
585 ///
586 /// \par RSSI
587 ///
588 /// The RSSI (Received Signal Strength Indicator) is measured and latched after the message sync bytes are received.
589 /// The latched RSSI is available from the lastRssi() member functionafter the complete message is received.
590 /// Although lastRssi()
591 /// supposedly returns a signed integer, in the case of this radio it actually returns an unsigned 8 bit integer (uint8_t)
592 /// and you will have to cast the return value to use it:
593 /// \code
594 /// uint8_t lastRssi = (uint8_t)rf24.lastRssi();
595 /// \endcode
596 /// The units of RSSI are arbitrary and relative, with larger unsigned numbers indicating a stronger signal. Values up to 255
597 /// are seen with radios in close proximity to each other. Lower limit of receivable strength is about 70.
598 ///
599 /// \par Transmitter Power
600 ///
601 /// You can control the transmitter power on the RF24/25/26/27 transceiver
602 /// with the RH_RF24::setTxPower() function. The argument can be any of
603 /// 0x00 to 0x4f (for RFM24/Si4460) or
604 /// 0x00 to 0x7f (for others)
605 /// 0x00 will yield no measurable power. For other settings there is a non-linear correlation with actual
606 /// RF power output (see below)
607 /// The default is 0x10. Eg:
608 /// \code
609 /// driver.setTxPower(0x10);
610 /// \endcode
611 ///
612 /// We have made some actual power measurements against
613 /// programmed power
614 /// - Anarduino Mini with RFM24-433 and RFM26-433 at Vcc = 3.3V, in CW mode, 434MHz
615 /// - 10cm RG58C/U soldered direct to RFM69 module ANT and GND
616 /// - bnc connecteor
617 /// - 12dB attenuator
618 /// - BNC-SMA adapter
619 /// - MiniKits AD8307 HF/VHF Power Head (calibrated against Rohde&Schwartz 806.2020 test set)
620 /// - Digitech QM-1460 digital multimeter
621 /// \code
622 /// Program power Measured Power dBm
