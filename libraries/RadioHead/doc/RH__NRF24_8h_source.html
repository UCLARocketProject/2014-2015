156 /// \class RH_NRF24 RH_NRF24.h <RH_NRF24.h>
157 /// \brief Send and receive addressed, reliable, acknowledged datagrams by nRF24L01 and compatible transceivers.
158 ///
159 /// Supported transceivers include:
160 /// - Nordic nRF24 based 2.4GHz radio modules, such as nRF24L01 http://www.nordicsemi.com/eng/Products/2.4GHz-RF/nRF24L01
161 /// and other compatible transceivers.
162 /// - Sparkfun WRL-00691 module with nRF24L01 https://www.sparkfun.com/products/691
163 /// or WRL-00705 https://www.sparkfun.com/products/705 etc.
164 /// - Hope-RF RFM73 http://www.hoperf.com/rf/2.4g_module/RFM73.htm and
165 /// http://www.anarduino.com/details.jsp?pid=121
166 /// and compatible devices (such as BK2423). nRF24L01 and RFM73 can interoperate
167 /// with each other.
168 ///
169 /// This base class provides basic functions for sending and receiving unaddressed, unreliable datagrams
170 /// of arbitrary length to 28 octets per packet. Use one of the Manager classes to get addressing and
171 /// acknowledgement reliability, routing, meshes etc.
172 ///
173 /// The nRF24 transceiver is configured to use Enhanced Shockburst with no acknowledgement and no retransmits.
174 /// TX_ADDR and RX_ADDR_P0 are set to the network address. If you need the low level auto-acknowledgement
175 /// feature supported by this chip, you can use our original NRF24 library
176 /// at http://www.airspayce.com/mikem/arduino/NRF24
177 ///
178 /// Naturally, for any 2 radios to communicate that must be configured to use the same frequency and
179 /// data rate, and with identical network addresses.
180 ///
181 /// The nRF24L01 (http://www.sparkfun.com/datasheets/Wireless/Nordic/nRF24L01P_Product_Specification_1_0.pdf)
182 /// is a low-cost 2.4GHz ISM transceiver module. It supports a number of channel frequencies in the 2.4GHz band
183 /// and a range of data rates.
184 ///
185 /// This library provides functions for sending and receiving messages of up to 28 octets on any
186 /// frequency supported by the nRF24L01, at a selected data rate.
187 ///
188 /// Several nRF24L01 modules can be connected to an Arduino, permitting the construction of translators
189 /// and frequency changers, etc.
190 ///
191 /// Example Arduino programs are included to show the main modes of use.
192 ///
193 /// \par Packet Format
194 ///
195 /// All messages sent and received by this class conform to this packet format, as specified by
196 /// the nRF24L01 product specification:
197 ///
198 /// - 1 octets PREAMBLE
199 /// - 4 octets NETWORK ADDRESS
200 /// - 9 bits packet control field
201 /// - 0 to 32 octets PAYLOAD, consisting of:
202 /// - 1 octet TO header
203 /// - 1 octet FROM header
204 /// - 1 octet ID header
205 /// - 1 octet FLAGS header
206 /// - 0 to 28 octets of user message
207 /// - 2 octets CRC
208 ///
209 /// \par Connecting nRF24L01 to Arduino
210 ///
211 /// The electrical connection between the nRF24L01 and the Arduino require 3.3V, the 3 x SPI pins (SCK, SDI, SDO),
212 /// a Chip Enable pin and a Slave Select pin.
213 /// If you are using the Sparkfun WRL-00691 module, it has a voltage regulator on board and
214 /// can be should with 5V VCC if possible.
215 /// The examples below assume the Sparkfun WRL-00691 module
216 ///
217 /// Connect the nRF24L01 to most Arduino's like this (Caution, Arduino Mega has different pins for SPI,
218 /// see below). Use these same connections for Teensy 3.1 (use 3.3V not 5V Vcc).
219 /// \code
220 /// Arduino Sparkfun WRL-00691
221 /// 5V-----------VCC (3.3V to 7V in)
222 /// pin D8-----------CE (chip enable in)
223 /// SS pin D10----------CSN (chip select in)
224 /// SCK pin D13----------SCK (SPI clock in)
225 /// MOSI pin D11----------SDI (SPI Data in)
226 /// MISO pin D12----------SDO (SPI data out)
227 /// IRQ (Interrupt output, not connected)
228 /// GND----------GND (ground in)
229 /// \endcode
230 ///
231 /// For an Arduino Leonardo (the SPI pins do not come out on the Digital pins as for normal Arduino, but only
232 /// appear on the ICSP header)
233 /// \code
234 /// Leonardo Sparkfun WRL-00691
235 /// 5V-----------VCC (3.3V to 7V in)
236 /// pin D8-----------CE (chip enable in)
237 /// SS pin D10----------CSN (chip select in)
238 /// SCK ICSP pin 3----------SCK (SPI clock in)
239 /// MOSI ICSP pin 4----------SDI (SPI Data in)
240 /// MISO ICSP pin 1----------SDO (SPI data out)
241 /// IRQ (Interrupt output, not connected)
242 /// GND----------GND (ground in)
243 /// \endcode
244 /// and initialise the NRF24 object like this to explicitly set the SS pin
245 /// NRF24 nrf24(8, 10);
246 ///
247 /// For an Arduino Mega:
248 /// \code
249 /// Mega Sparkfun WRL-00691
250 /// 5V-----------VCC (3.3V to 7V in)
251 /// pin D8-----------CE (chip enable in)
252 /// SS pin D53----------CSN (chip select in)
253 /// SCK pin D52----------SCK (SPI clock in)
254 /// MOSI pin D51----------SDI (SPI Data in)
255 /// MISO pin D50----------SDO (SPI data out)
256 /// IRQ (Interrupt output, not connected)
257 /// GND----------GND (ground in)
258 /// \endcode
259 /// and you can then use the constructor RH_NRF24(8, 53).
260 ///
261 /// For an Itead Studio IBoard Pro http://imall.iteadstudio.com/iboard-pro.html, connected by hardware SPI to the
262 /// ITDB02 Parallel LCD Module Interface pins:
263 /// \code
264 /// IBoard Signal=ITDB02 pin Sparkfun WRL-00691
265 /// 3.3V 37-----------VCC (3.3V to 7V in)
266 /// D2 28-----------CE (chip enable in)
267 /// D29 27----------CSN (chip select in)
268 /// SCK D52 32----------SCK (SPI clock in)
269 /// MOSI D51 34----------SDI (SPI Data in)
270 /// MISO D50 30----------SDO (SPI data out)
271 /// IRQ (Interrupt output, not connected)
272 /// GND 39----------GND (ground in)
273 /// \endcode
274 /// And initialise like this:
275 /// \code
276 /// RH_NRF24 nrf24(2, 29);
277 /// \endcode
278 ///
279 /// For an Itead Studio IBoard Pro http://imall.iteadstudio.com/iboard-pro.html, connected by software SPI to the
280 /// nRF24L01+ Module Interface pins. CAUTION: performance of software SPI is very slow and is not
281 /// compatible with other modules running hardware SPI.
282 /// \code
283 /// IBoard Signal=Module pin Sparkfun WRL-00691
284 /// 3.3V 2-----------VCC (3.3V to 7V in)
285 /// D12 3-----------CE (chip enable in)
286 /// D29 4----------CSN (chip select in)
287 /// D9 5----------SCK (SPI clock in)
288 /// D8 6----------SDI (SPI Data in)
289 /// D7 7----------SDO (SPI data out)
290 /// IRQ (Interrupt output, not connected)
291 /// GND 1----------GND (ground in)
292 /// \endcode
293 /// And initialise like this:
294 /// \code
295 /// #include <SPI.h>
296 /// #include <RH_NRF24.h>
297 /// #include <RHSoftwareSPI.h>
298 /// Singleton instance of the radio driver
299 /// RHSoftwareSPI spi;
300 /// RH_NRF24 nrf24(12, 11, spi);
301 /// void setup() {
302 /// spi.setPins(7, 8, 9);
303 /// ....
304 /// \endcode
305 ///
306 /// You can override the default settings for the CSN and CE pins
307 /// in the NRF24() constructor if you wish to connect the slave select CSN to other than the normal one for your
308 /// Arduino (D10 for Diecimila, Uno etc and D53 for Mega)
309 ///
310 /// Caution: on some Arduinos such as the Mega 2560, if you set the slave select pin to be other than the usual SS
311 /// pin (D53 on Mega 2560), you may need to set the usual SS pin to be an output to force the Arduino into SPI
312 /// master mode.
313 ///
314 /// Caution: this module has not been proved to work with Leonardo, at least without level
315 /// shifters between the nRF24 and the Leonardo. Tests seem to indicate that such level shifters would be required
316 /// with Leonardo to make it work.
317 ///
318 /// It is possible to have 2 radios conected to one arduino, provided each radio has its own
319 /// CSN and CE line (SCK, SDI and SDO are common to both radios)
320 ///
321 /// \par SPI Interface
322 ///
323 /// You can interface to nRF24L01 with with hardware or software SPI. Use of software SPI with the RHSoftwareSPI
324 /// class depends on a fast enough processor and digitalOut() functions to achieve a high enough SPI bus frequency.
325 /// If you observe reliable behaviour with the default hardware SPI RHHardwareSPI, but unreliable behaviour
326 /// with Software SPI RHSoftwareSPI, it may be due to slow CPU performance.
327 ///
328 /// Initialisation example with hardware SPI
329 /// \code
330 /// #include <RH_NRF24.h>
331 /// RH_NRF24 driver;
332 /// RHReliableDatagram manager(driver, CLIENT_ADDRESS);
333 /// \endcode
334 ///
335 /// Initialisation example with software SPI
336 /// \code
337 /// #include <RH_NRF24.h>
338 /// #include <RHSoftwareSPI.h>
339 /// RHSoftwareSPI spi;
340 /// RH_NRF24 driver(8, 10, spi);
341 /// RHReliableDatagram manager(driver, CLIENT_ADDRESS);
342 /// \endcode
343 ///
344 /// \par Example programs
345 ///
346 /// Several example programs are provided.
347 ///
348 /// \par Radio Performance
349 ///
350 /// Frequency accuracy may be debatable. For nominal frequency of 2401.000 MHz (ie channel 1),
351 /// my Yaesu VR-5000 receiver indicated the center frequency for my test radios
352 /// was 2401.121 MHz. Its not clear to me if the Yaesu
353 /// is the source of the error, but I tend to believe it, which would make the nRF24l01 frequency out by 121kHz.
354 ///
355 /// \par Radio operating strategy and defaults
356 ///
357 /// The radio is enabled all the time, and switched between TX and RX modes depending on
358 /// whether there is any data to send. Sending data sets the radio to TX mode.
359 /// After data is sent, the radio automatically returns to Standby II mode. Calling waitAvailable() or
360 /// waitAvailableTimeout() starts the radio in RX mode.
361 ///
362 /// The radio is configured by default to Channel 2, 2Mbps, 0dBm power, 5 bytes address, payload width 1, CRC enabled
363 /// 2 byte CRC, No Auto-Ack mode. Enhanced shockburst is used.
364 /// TX and P0 are set to the Network address. Node addresses and decoding are handled with the RH_NRF24 module.
365 ///
366 /// \par Memory
367 ///
368 /// Memory usage of this class is minimal. The compiled client and server sketches are about 6000 bytes on Arduino.
369 /// The reliable client and server sketches compile to about 8500 bytes on Arduino.
370 /// RAM requirements are minimal.
371 ///
